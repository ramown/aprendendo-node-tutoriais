# Guia de Contribuição

Obrigado por contribuir com este repositório.

## Como reportar bugs

Abra uma issue usando o template de bug e informe:

- contexto do problema
- comportamento esperado
- comportamento atual
- passos para reproduzir
- versão do Node.js
- sistema operacional

## Como sugerir funcionalidades

Abra uma issue usando o template de feature request e descreva:

- problema que a funcionalidade resolve
- proposta de solução
- alternativas consideradas
- impacto esperado

## Padrões de commit

Sugestão de padrão:

- `feat: adiciona novo tutorial`
- `fix: corrige instrução de README`
- `docs: atualiza roadmap`
- `chore: ajusta configuração do projeto`

## Processo de Pull Request

1. Faça um fork do projeto
2. Crie uma branch descritiva
3. Faça alterações pequenas e objetivas
4. Atualize a documentação quando necessário
5. Abra um pull request usando o template disponível

## Boas práticas

- mantenha consistência na estrutura de pastas
- não adicione código de tutorial fora do local correto
- prefira mudanças incrementais
- revise ortografia e clareza dos textos
